# План проекта bot-creter — верхнеуровневые заметки

Этот файл — не пошаговый план, а место для коротких, долгоживущих маркеров о направлении проекта, которые не стоит терять между сессиями/фазами. Детали — в `docs/STAGE2_DESIGN.md` и `STAGE2_REPORT.md`.

- **`tour_operator` — временная монолитная форма, не конечный продукт.** Задачу туроператора в финале решает стадия «Офисы» (см. ниже): монолит будет разобран на компоненты или станет прообразом CRM-части «офиса». Не мультиплицировать `tour_operator` как есть — не строить многотенантную веб-архитектуру ради него.
- **Стадия «Офисы» (будущая, не детализирована).** Связка из нескольких ботов + оркестратор + общая CRM, которую собирает бот-создатель — для сценариев вроде туроператора, где одному «бизнесу» нужно больше одного бота и общее хранилище данных/интерфейс поверх них. Детализировать по достижении этой стадии.
